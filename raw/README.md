# raw/ — 不可变原始来源

> 这里是你的**原始知识来源**的只读存档。Cub 从这里读取、摄入到 wiki，但**绝不修改**这里的文件。

## 目录说明

| 目录 | 用途 | 示例 |
|------|------|------|
| `clippings/` | Web Clipper 剪藏 | 网页文章、推文线程 |
| `articles/` | 完整文章 | 博客文章、深度报道 |
| `papers/` | 学术论文 | arXiv 论文、研究报告 |
| `books/` | 书籍笔记 | 按章整理的读书笔记 |
| `podcasts/` | 播客记录 | 播客摘要和关键观点 |
| `projects/` | 项目复盘 | 自己项目的设计文档、复盘记录 |
| `notes/` | 学习笔记 | 课程笔记、会议记录 |
| `repos/` | 外部仓库 | `git clone` 的外部 GitHub 仓库（Cub 读代码库来源） |
| `assets/` | 附件 | 图片、截图、PDF |

## 使用方式

1. 将新来源文件放入对应目录
2. 告诉 Cub："摄入 `raw/articles/xxx.md`"
3. Cub 读取 → 写 source 摘要 → 更新相关概念页 → 更新 index

## 原则

- **不可变**：放入 raw/ 后不再修改——它是"真相的原始记录"
- **Cub 只读**：Cub 读取但不写入此目录
- **来源可追溯**：wiki 中的每条声明必须能追溯到 raw/ 中的具体来源
