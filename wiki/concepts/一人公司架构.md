# 一人公司架构

> **AI 原生一人公司的完整架构**：知识层 + Agent 层 + 执行层 + 基础设施层的四层模型。

## 四层架构模型

```
┌─────────────────────────────────────────────────────┐
│                   Layer 4: 界面层                      │
│   Obsidian · Web UI · Claude Code · Chrome Clipper    │
├─────────────────────────────────────────────────────┤
│                   Layer 3: 执行层                      │
│  AI IDE (Cursor/Windsurf) · No-Code (Bolt.new)       │
│  BaaS (Supabase) · 部署 (Vercel/Zeabur)              │
│  收款 (Lemon Squeezy) · 营销 (Screen Studio)          │
│  自动化 (Make.com) · 客服 (Crisp)                     │
├─────────────────────────────────────────────────────┤
│                   Layer 2: Agent 层                    │
│  AI Agent 团队 (5 Worker + 1 Cub)                     │
│  定时任务 · Skills 市场 · Missions & Tasks            │
│  强制引用 · 质量闸门                                   │
├─────────────────────────────────────────────────────┤
│                   Layer 1: 知识层                      │
│  Raw Sources (不可变) → Wiki (LLM 维护)                │
│  知识图谱 · 版本控制 (Git)                              │
│  CLAUDE.md (宪法) + index.md (路由器)                  │
└─────────────────────────────────────────────────────┘
```

### Layer 1: 知识层 —— 公司的长期记忆

**核心原则**：知识编译一次，持续保鲜。不做每次查询时从零检索（那是 RAG），而是增量构建持久化的结构化 wiki。

关键组件：
- **Raw Sources**（不可变）：原始文档的只读存档，按类型分目录
- **Wiki**（LLM 维护）：实体、概念、来源摘要、对比、查询归档——自动交叉引用
- **版本控制**：Git 记录每一次变更，可追溯到任意时间点

### Layer 2: Agent 层 —— 公司的虚拟团队

**核心原则**：多 Agent 不需要框架——单程序多窗口，各自读各自的 CLAUDE.md。Worker 之间不直接通信，通过文件系统协作。

当前已落地的五个 Worker 角色，按产品周期排列：

| Worker | 职责 | 核心约束 |
|--------|------|---------|
| **Scout** | 市场调研、竞品追踪、用户发现 | 调研结果写入 `briefs/scout→designer.md` |
| **Designer** | 产品设计、交互流程、视觉设计 | 不向外搜索；拒绝通用 AI 美学 |
| **Builder** | 编码、调试、端到端验证、CLI 参考 | `src/` 可写，`src/tests/` 只读；写 `builder→tester.md` |
| **Tester** | 全流程 E2E 测试、产出 fix-prompt、示例用法 | 不看代码写测试；一个 `e2e.test.js`；`src/tests/` 可写 |
| **Seller** | 封版清理、打包发布、README/快速入门/Landing Page、分发运营 | 读 Scout + Builder + Tester；对外第一责任人 |

**Cub** 是第 6 个角色，但不是 Worker——它是公司记忆的唯一维护者，负责 wiki 的收录、索引和健康检查。

**四条共享通道**（详见 [[concepts/多 Agent 协作模型]]）：
- `src/` — 共享代码，含目录级权限边界
- `wiki/` — 共享记忆，Cub 维护，所有 Worker 只读
- `inboxes/` — 各自吐经验，Cub 整合后进入 wiki
- `briefs/` — 阶段交接文档，上游 Worker 写入、下游 Worker 只读

**Worker 部署模型**：`agents/templates/` 存放角色模板（模板之间不互相引用），复制 + 填入项目名/描述/技术栈 = Worker 上岗。新增角色不影响已有模板。

### Layer 3: 执行层 —— 从灵感到收款

**核心原则**：2025 年，AI IDE + No-Code + BaaS 让一个人活成一支队伍。

| 阶段 | 核心工具 | 关键理念 |
|------|----------|----------|
| **验证** | Perplexity, GummySearch, Carrd | 写代码前先确认有人买单 |
| **开发** | Cursor, Windsurf, Bolt.new | MVP 从"周"缩短到"小时" |
| **基础设施** | Supabase, Vercel, Coolify | 零运维、免费起步、按量付费 |
| **设计** | Recraft, Napkin.ai | 一人做出专业团队的外观 |
| **收款** | Lemon Squeezy (MoR) | 让平台替你处理全球税务 |
| **营销** | Screen Studio, Typefully | Build in Public，获取自然流量 |
| **自动化** | Make.com | 付款→欢迎邮件→Discord→数据库，全自动 |

### Layer 4: 界面层 —— 人与 AI 的交互界面

**核心原则**：同一文件系统，多种交互方式——桌面端、浏览器、CLI、Obsidian 图视图。

关键组件：
- **Obsidian**：本地 Markdown 编辑器 + 图视图，可视化 wiki 连接结构
- **Claude Code**：AI 原生交互——通过自然语言操作知识库
- **Chrome Web Clipper**：一键捕获网页 + 自动摄入

## 架构哲学

### 1. 去平台锁定（De-Platform Lock-in）

- 你的 AI 记忆不应绑定在任何单一供应商
- 本地 Markdown 文件 + Git = 终极可移植性
- "AI 工具会不断变化，但你的 Vault 永远跟你在一起"

### 2. 知识复利（Knowledge Compounding）

- 每次摄入不是孤立的操作——它交叉更新 10-15 个 wiki 页面
- 版本控制让知识演进可审计："这个结论是基于什么来源？何时做出的？"

### 3. 预索引压倒运行时检索

参见 [[concepts/预索引 vs 运行时扫描]]。一人公司场景下，预索引的优势更加明显：
- 创始人的决策频率远高于新信息来源的频率
- 每次决策都需要完整的知识上下文——不能每次从零检索

### 4. AI 角色升维

Cub 的定位：不是跟随指令的工具，而是积累你全部决策上下文、与你讨论想法、挑战你的思考、帮你做出更好判断的**长期合伙人**。

## 最小可行架构（MVA）

如果从零开始搭建一人公司 AI 基础设施，建议的最小集合：

```
我的公司/                       ← 一个文件夹 = 整个公司
├── CLAUDE.md                # Schema 层：Cub 的宪法
├── TASKS.md                 # 公司基础设施待办
├── .claude/                 # Claude Code 专属配置
├── tools/                   # 可复用工具脚本
├── raw/                     # 不可变原始来源
├── wiki/                    # 长期记忆（Cub 唯一写者）
│   ├── index.md             #   路由器
│   ├── overview.md          #   全局图景
│   ├── log.md               #   时间线日志
│   ├── entities/            #   实体
│   ├── concepts/            #   概念
│   ├── sources/             #   来源摘要
│   ├── comparisons/         #   对比分析
│   └── queries/             #   查询归档
├── agents/                  # Agent 模板
│   ├── cub.md               #   Cub persona
│   └── templates/           #   Worker 角色模板
├── inboxes/                 # Worker 踩坑收件箱
├── projects/                # 进行中的项目
└── archive/                 # 已封存项目
```

**原则**：零数据库、零安装脚本、零供应商锁定。所有数据是纯文本 Markdown 文件。详见 [[concepts/硬盘即公司]]。
