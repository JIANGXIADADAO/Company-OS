## 核心论点

### 1. 硬盘即公司：去平台锁定的物理终极形态
整个一人公司就是移动硬盘上的一个文件夹。零数据库、零安装脚本、零供应商锁定。拔掉硬盘 = 公司下班。参见 [[concepts/硬盘即公司]]。

### 2. index.md 是路由器，不是 Agent
Agent 启动时加载的导航地图。始终在场，目视导航——不需要单独的"记忆分配 Agent"。参见 [[concepts/Agent 记忆分级系统]]。

### 3. 记忆按加载优先级分为五级
宪法级（CLAUDE.md）→ 地图级（index.md + overview.md）→ 领域级（wiki 页面）→ 过程级（断点标记）→ 归档级（旧版本封存）。启动成本恒定，不随知识库增长而膨胀。参见 [[concepts/Agent 记忆分级系统]]。

### 4. 读是自由的，写是单点的
Worker 可以自由读 wiki（路由 → 导航 → 深入），但只有 Cub 能写。Git 处理文件冲突，处理不了语义冲突。参见 [[concepts/一人公司架构]]。

### 5. 自学习循环：Worker 踩坑 → inbox → Cub → wiki → 变聪明
经验按验证等级升迁：待验证 → 多次观察确认 → 已知局限。参见 [[concepts/Agent 自学习循环]]。

### 6. 预索引压倒运行时检索
当查询次数远超索引次数时，预索引是压倒性更优策略。瓶颈不是 access（能不能访问到信息），是 structure（能不能把问题映射到正确的实体）。参见 [[concepts/预索引 vs 运行时扫描]]。

### 7. 多 Agent 不需要框架——单程序多窗口，各自读各自的 CLAUDE.md
Worker 之间不直接通信，通过四条共享通道协作：`src/`（共享代码）、`wiki/`（共享记忆）、`inboxes/`（各自吐经验）、`briefs/`（阶段交接文档）。参见 [[concepts/多 Agent 协作模型]]。
